## Code Modification Rules

- **Phased Implementation:** Always follow the phase milestones outlined in `.meta/tasks/index.md`.
- **Commit Format:** Use conventional commits in lowercase (e.g., `feat: add product page offer render`).
- **Data Integrity:** Keep structural changes isolated. Never mix application modifications and documentation changes in single commits.
